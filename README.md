# PR Review Service

### Стек:
 - Golang:1.25.4
 - Gin
 - Postgresql:15

### Запуск проекта

#### 1. Запуск docker-compose:
```bash
make up
```

#### 2. Для остановки:
```bash
make down
```

### Дополнительные команды для работы с проектом:

#### 1. Просмотр логов контейнера приложения:
```bash
make logs
```

#### 2. Отдельное создание контейнера приложения:
```bash
make docker-build
```

### Конфигурация линтера

Конфигурация находится в файле `.golangci.yml`

#### В проекте включены основные линтеры:

- **govet** - Проверяет типичные ошибки Go-кода: неправильные теги JSON, подозрительные конструкции, неправильные вызовы форматирования и т.д.
- **revive** - Проверяет стиль кода, формат, названия, структуру.
- **errcheck** - Проверяет, что обрабатываются ошибки, а не игнорируются.
- **staticcheck** - Находит логические ошибки, устаревший код, неправильное использование стандартных функций, потенциальные паники.
- **ineffassign** - Находит переменные, которым присвоено значение, но это значение никогда не используется.
- **gocyclo** - Проверяет цикломатическую сложность - измеряет, насколько логически сложна функция.
- **unparam** - Находит неиспользуемые параметры функции.
- **unused** - Находит неиспользуемые переменные, константы, функции и т.д.

#### Настройки вывода ошибок:

- **max-issues-per-linter: 0** - Линтер покажет всё, что нашёл.
- **max-same-issues: 0** - Можно увидеть все одинаковые повторяющиеся проблемы.
- **new: false** - Не ограничивает проверку только новыми ошибками. Проверяет весь проект каждый раз.
- **fix: false** - Линтер не будет автоматически исправлять ошибки.

#### Запуск линтера:
```bash
make lint
```

### Дополнительные endpoints:

#### 1. Получает статистику: сколько раз каждого пользователя назначали ревьюером:

```
GET /stats/reviewers
```

#### 2. Получает статистику: количество назначенных ревьюеров для каждого pull request

```
GET /stats/pull_request
```
